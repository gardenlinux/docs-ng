# Customization Guides

Learn how to build custom Garden Linux images with your own features and flavors.

## What You'll Find Here

This section covers:

- **Building custom features** - Add specific capabilities to Garden Linux
- **Building custom flavors** - Create pre-configured images for specific use cases
- **Testing your builds** - Validate custom images before deployment

## Guides

- [Building Features](/how-to/customization/building-features) - How to create custom Garden Linux features
- [Building Flavors](/how-to/customization/building-flavors) - How to create custom Garden Linux flavors
- [Testing Builds](/how-to/customization/testing-builds) - How to test custom Garden Linux builds

## Prerequisites

Before customizing Garden Linux, you should:

1. Understand [Flavors and Features](/explanation/flavors-and-features) concepts
2. Have a working development environment (see [Contributing](/contributing/dev-environment))
3. Be familiar with the [Architecture](/explanation/architecture)
